# ProjectX Bot 🚀
## Production-ready Telegram бот для управления защищенными подключениями Amnezia
**ProjectX Bot** — это современный, отказоустойчивый Telegram-бот на базе `aiogram 3.x` для продажи и автоматизированного управления подключениями через HTTP-интерфейс Amnezia API. Поддерживает подключение неограниченного количества серверов, нативную оплату через Telegram Stars, реферальную систему, автоматический мониторинг трафика и гибкое администрирование.

---

## ✨ Возможности
* 🌍 **Мульти-VPS архитектура:** Подключение неограниченного количества серверов Amnezia через админ-панель с автоматической валидацией локации, проверкой поддерживаемых протоколов и свободным выбором сервера пользователем при создании устройств.
* 💳 **Платежные шлюзы:** Нативная поддержка инвойсов Telegram Stars (`XTR`) и встроенная СБП-заглушка мгновенного зачисления для быстрой интеграции фиатного эквайринга.
* 🎁 **Реферальная система:** Автоматическое отслеживание пригласивших пользователей по deep-link параметрам (`/start?start=ref_ID`). Начисление бонусных дней рефереру происходит автоматически сразу после первой успешной оплаты приглашенного друга.
* 📱 **Управление устройствами:** Полноценный селф-сервис для пользователей. Генерация уникальных обфусцированных конфигурационных ключей доступа, динамическая сборка и скачивание готовых файлов формата `.conf` для нативного клиента Amnezia.
* 🔐 **Промышленная безопасность:** Двустороннее симметричное шифрование конфиденциальных полей базы данных (`raw_config`, `peer_id` для профилей и `api_key` для серверов) алгоритмом Fernet (`cryptography`) "из коробки". Защита от запуска при некорректном или отсутствующем ключе шифрования.
* 🤖 **Комплекс фоновых воркеров (`asyncio` задачи):**
  * `traffic`: Синхронизация потребленного трафика (Download/Upload) и времени последней активности пиров с Amnezia API каждые 15 минут.
  * `cleanup`: Ежедневный автоматический аудит и вычищение "призрачных пиров" (Dangling Peers) на серверах, удаленных локально из БД, исключая рассинхронизацию состояний.
  * `notifications`: Опросы каждые 30 минут для заблаговременного уведомления пользователей об истечении подписки (за 3 дня, 1 день и 2 часа).
  * `payments`: Автоматический трекинг зависших инвойсов (в статусе pending более 1 часа) с отправкой алертов администраторам.
* 🛠 **Многофункциональная админ-панель:**
  * Живая статистика дашборда (всего юзеров, активные подписки, новые за 24 часа, емкость свободных IP).
  * Управление пользователями (поиск по ID, выдача бесконечного или кастомного доступа, просмотр подключенных устройств, жесткий бан с моментальным аннулированием сессий на Amnezia API).
  * Конструктор тарифов и серверов.
  * Система логирования действий администраторов (Аудит-лог).
  * HTML-рассылки с поддержкой медиафайлов/документов, контролем флуда и автоматической отметкой заблокировавших бота аккаунтов.
* ⚡ **Оптимизация:** Сверхлегкий конвейер middleware (`DBSessionMiddleware` для инъекций асинхронных сессий, `UserContextMiddleware` для кэширования контекста и моментального отсечения забаненных пользователей, `ThrottlingMiddleware` на базе `cachetools` для защиты от флуда кнопками).

## 🚀 Быстрый старт
### Локальный запуск (Разработка):
1. Клонируйте репозиторий и перейдите в корень:
```bash
git clone https://github.com/justik13/projectx.git
cd projectx
```
2. Разверните и активируйте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```
3. Установите необходимые пакеты:
```bash
pip install -r requirements.txt
```
4. Создайте файл конфигурации `.env` на основе примера ниже, обязательно сгенерировав 32-байтный base64 ключ шифрования:
```bash
python -c "from cryptography.fernet import Fernet; print(f'DB_ENCRYPTION_KEY={Fernet.generate_key().decode()}')"
```
5. Запустите бота:
```bash
python -m bot.main
```

### Production Deploy (Ubuntu/Debian):
Для автоматического развертывания проекта в изолированном production-окружении выполните команду:
```bash
sudo bash deploy.sh
```
**Скрипт `deploy.sh` полностью автономен и осуществляет:**
* Проверку совместимости ОС и установку всех системных бинарников (`sqlite3`, `python3-venv`, `rsync` и др.).
* Создание непривилегированного системного пользователя `projectx` для изоляции процесса.
* Синхронизацию файлов в директорию `/opt/projectx-bot` с выставлением безопасных прав доступа.
* Интерактивный конфигуратор параметров `.env` с автоматической генерацией Fernet-ключа.
* Первичную накатку структуры БД, оптимизацию SQLite (включение `WAL`, `synchronous=NORMAL`, настройку кэша).
* Регистрацию, изоляцию и запуск фонового демона **systemd** (`projectx-bot.service`) с политикой автоперезапуска.
* Настройку ежедневных атомарных бэкапов базы данных по `cron` в 3:00 (с ротацией файлов за 30 дней).
* Внедрение скрипта пятиминутного **Healthcheck-мониторинга** (при падении бота скрипт перезапустит службу и отправит уведомление первому админу через Telegram API).
* Конфигурирование утилиты `logrotate` для системных логов.

## ⚙️ Конфигурация (.env)
Файл конфигурации должен располагаться в корне рабочей директории. Права доступа на сервере автоматически ограничиваются до `600`.

| Переменная | Обязательная | Значение по умолчанию | Описание |
| --- | --- | --- | --- |
| `BOT_TOKEN` | **Да** | — | Токен целевого бота от @BotFather |
| `ADMIN_IDS` | **Да** | — | Числовые Telegram ID администраторов через запятую (например: `1234,5678`) |
| `DB_PATH` | Нет | `./bot_data.db` | Относительный или абсолютный путь к файлу SQLite |
| `DB_ENCRYPTION_KEY` | **Да** | — | 32-байтный base64 Fernet ключ шифрования чувствительных полей |
| `DEFAULT_DEVICE_LIMIT` | Нет | `2` | Количество доступных устройств для нового пользователя |
| `REFERRAL_BONUS_DAYS` | Нет | `3` | Количество дней подписки, начисляемых рефереру за оплату друга |
| `SUPPORT_USERNAME` | Нет | `@support_username` | Юзернейм аккаунта техподдержки в интерфейсе бота |

## 🔌 Спецификация интеграции с Amnezia API
Бот настроен на жесткое взаимодействие с эндпоинтами Amnezia API (протокол `amneziawg2`). При добавлении сервера в админке, бот осуществляет вызовы:
1. `GET /healthz` — Проверка доступности API-ноды.
2. `GET /server` — Получение поддерживаемых сервером протоколов (обязательно наличие `amneziawg2`) и лимита `maxPeers`.
3. `POST /clients` — Создание нового пира. Передается сгенерированное имя клиента и временная метка `expiresAt` (или `null`). Возвращает `id` (peer_id) и `config` (строка `vpn://`).
4. `DELETE /clients` — Полное безвозвратное удаление пира с сервера при уничтожении устройства пользователем.
5. `PATCH /clients` — Удаленное изменение статуса пира (`status="disabled"` при бане пользователя или остановке сервера админом, `status="active"` при разблокировке / продлении доступа).
6. `GET /clients` — Постраничный сбор метрик потребленного трафика для фонового воркера.

## 🛡️ Безопасность и Отказоустойчивость
* **Паттерн компенсирующих транзакций:** Если в процессе создания устройства в Telegram Amnezia API успешно сгенерировал ключ, но произошел сбой при записи в локальную базу данных бота, сессия не зависнет на сервере — бот перехватит исключение и выполнит автоматический компенсирующий `DELETE`-запрос к API.
* **Изоляция сбоев API:** При удалении устройства пользователем, если Amnezia API временно недоступен, бот прерывает транзакцию и выдает предупреждение, не удаляя профиль из локальной БД. Это исключает появление "зависших" (Dangling) неудаляемых пиров на удаленном сервере.
* **Защита от инъекций:** Весь кастомный ввод пользователей (имена устройств, юзернеймы, имена профилей) проходит через единую санитизирующую обертку `utils.telegram.safe()` с принудительным HTML-экранированием перед рендером в сообщениях или админ-панели.
* **Безопасная деинсталляция:** Скрипт `uninstall.sh` предоставляет гибкое меню удаления системы. Поддерживает как тотальную зачистку, так и режим миграции (когда база данных и `.env` безопасно упаковываются в архив `/root/projectx-backup-TIMESTAMP`, а системный юзер, cron-задачи, systemd-конфиги и скрипты автоматизации корректно вычищаются из ОС).
