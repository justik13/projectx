# AmneziaVPN Telegram Bot

Telegram-бот для управления VPN-профилями AmneziaVPN с поддержкой мультипрофилей, секретных ключей, короткиих ссылок, веб-сервиса и Telegram Mini App на основе AmneziaApi

## Возможности

- 🔐 Мультипрофили — каждый пользователь может иметь несколько VPN-профилей
- 📥 Получение конфигурационного файла (строка `vpn://` и `.vpn`-файл)
- 👤 Просмотр статуса каждого профиля (онлайн, трафик, последнее подключение)
- 🗑 Удаление собственных профилей
- 🖥 Статус сервера
- 🔔 Уведомления администратора при создании/удалении профиля
- ⚙️ Режим «только для администраторов» (`BOT_MODE=admin`)
- 🔑 Секретные ключи — пользователь получает одноразовый ключ для самостоятельного создания профиля через веб-сервис
- 🔗 Короткие ссылки — временные ссылки на конфигурацию (24 ч) через веб-сервис
- 🌐 Telegram Mini App — веб-интерфейс управления профилями прямо в Telegram
- 🛡 Защита от сканеров и ботов (`security.py`) — блокировка по IP, rate limiting, детектирование плохих User-Agent и путей

### Команды бота

| Команда | Описание |
|---|---|
| `/start`, `/menu` | Главное меню |
| `/mykey` | Показать текущий секретный ключ |
| `/newkey` | Создать новый секретный ключ (старый аннулируется) |

### Админ-панель

- 👥 Список пользователей с пагинацией (кликабельные ID → профиль в Telegram)
- 🃏 Карточка пользователя: все профили, трафик, IP, статус
- 🚫 Бан/разбан пользователей
- ⏸ Включение/отключение отдельных профилей
- 🗑 Удаление профилей с синхронизацией Amnezia API
- 📊 Статистика: трафик, топ пользователей, онлайн
- 🔍 Поиск по ID или имени профиля
- 📢 Рассылка сообщений всем пользователям
- 📋 Экспорт пользователей в CSV
- 🔎 Просмотр всех пиров Amnezia
- 🔑 Управление секретными ключами пользователей (блокировка/отзыв)

## Структура файлов

```
bot/
├── bot.py              # Главный файл: хендлеры, регистрация, middleware
├── admin_handlers.py   # Все хендлеры админ-панели
├── shared.py           # Общие утилиты, клавиатуры, хелперы
├── config.py           # Загрузка настроек из .env (pydantic-settings)
├── database.py         # Асинхронная SQLite база (aiosqlite + шифрование)
├── amnezia_client.py   # HTTP-клиент для Amnezia Admin API (aiohttp + retry)
├── web_service.py      # Веб-сервис: ввод ключа → получение конфига, короткие ссылки
├── miniapp.py          # Telegram Mini App (Flask)
├── security.py         # Защита Flask-приложений от сканеров и ботов
├── start.sh            # Скрипт запуска через screen
├── stop.sh             # Скрипт остановки screen-сессий
├── requirements.txt
├── .env
└── README.md
```

## Установка

```bash
# 1. Скопировать файлы бота
cd /root/bot

# 2. Создать виртуальное окружение
python3 -m venv me
source /root/me/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Настроить .env
nano .env
```

## Запуск

### Через screen (рекомендуется)

```bash
chmod +x /root/bot/start.sh /root/bot/stop.sh
/root/bot/start.sh
```

`start.sh` запускает три процесса в отдельных screen-сессиях:

```bash
screen -r bot      # подключиться к боту
screen -r miniapp  # подключиться к Mini App
screen -r webapp   # подключиться к веб-сервису
screen -ls         # список всех сессий
# Ctrl+A, D — отключиться от сессии (процесс продолжает работать)
```

Для остановки:

```bash
/root/bot/stop.sh
```

### Автозапуск при старте сервера

```bash
crontab -e
```

Добавить строку:

```
@reboot /root/bot/start.sh
```

### Вручную

```bash
source /root/me/bin/activate
cd /root/bot

python bot.py          # в одном терминале
python web_service.py  # во втором терминале
python miniapp.py      # в третьем терминале
```

## Правила именования профиля

- До **16 символов**
- Только буквы (латиница или кириллица) и цифры
- Имя глобально уникально

## API-эндпоинты Amnezia Admin API

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/clients` | Список всех клиентов и пиров |
| `POST` | `/clients` | Создание нового клиента |
| `PATCH` | `/clients` | Обновление статуса клиента |
| `DELETE` | `/clients` | Удаление клиента |
| `GET` | `/server` | Информация о сервере |
| `GET` | `/server/load` | Нагрузка (CPU, RAM, диск) |
| `GET` | `/healthz` | Healthcheck |

## Веб-сервис (`web_service.py`, порт 5001)

Публичный сайт для получения конфигурации по одноразовому секретному ключу, а также для раздачи конфигов по коротким ссылкам.

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/` | Страница ввода ключа и имени профиля |
| `POST` | `/connect` | Активация ключа и получение конфига |
| `GET` | `/c/<slug>` | Страница с конфигом по короткой ссылке |
| `GET` | `/api/ping` | Пинг до VPN-сервера (ICMP, серверная сторона) |
| `GET` | `/robots.txt` | Запрет индексации |

Эндпоинт `/connect` требует динамический токен сессии (`X-Dynamic-Token`), который генерируется при загрузке главной страницы и действует 5 минут.

## Mini App эндпоинты (`miniapp.py`, порт 5000)

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/` | Главная страница Mini App |
| `GET` | `/api/me` | Профили текущего пользователя |
| `POST` | `/api/create` | Создать профиль |
| `DELETE` | `/api/profile/<id>` | Удалить свой профиль |
| `GET` | `/api/config/<id>` | Получить конфиг профиля + короткую ссылку |
| `GET` | `/api/server` | Статус сервера (нагрузка, онлайн, пиры) |
| `GET` | `/api/ping` | Пинг до VPN-сервера |
| `GET` | `/api/mykey` | Получить (или создать) текущий секретный ключ |
| `POST` | `/api/newkey` | Создать новый секретный ключ (старый аннулируется) |
| `POST` | `/api/validate_hash` | Валидация Telegram initData |

Все эндпоинты, кроме `/api/ping` и `/api/validate_hash`, требуют авторизацию через заголовок `X-Telegram-Init-Data` (подпись Telegram WebApp).

## Схема базы данных

| Таблица | Назначение |
|---|---|
| `users` | Пользователи: `telegram_id`, `banned`, `key_blocked`, `created_at` |
| `vpn_profiles` | VPN-профили: `telegram_id`, `vpn_name`, `peer_id`, `raw_response`, `last_ip`, `disabled`, `via_key` |
| `secret_keys` | Одноразовые ключи: `telegram_id`, `key_value`, `used`, `revoked`, `can_create`, `used_at` |
| `short_links` | Короткие ссылки: `profile_id`, `slug` (живут 24 ч, чистятся автоматически) |

Чувствительные поля (`peer_id`, `raw_response`, `last_ip`) хранятся зашифрованными (Fernet). База поддерживает автоматическую миграцию схемы и миграцию из старой таблицы `vpn_users`.
