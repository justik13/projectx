## API Docs
- Платежи > Рекуррентные СБП-подписки [Создать подписку](https://docs.platega.io/%D1%81%D0%BE%D0%B7%D0%B4%D0%B0%D1%82%D1%8C-%D0%BF%D0%BE%D0%B4%D0%BF%D0%B8%D1%81%D0%BA%D1%83-40029698e0.md): Подписка — это регулярное автосписание с плательщика через СБП. Вы один раз создаёте подписку и отправляете плательщика на нашу платёжную форму; дальше привязку, активацию и все списания делаем мы, а вам приходят callback'и. Баланс пополняется по каждому успешному списанию.
- Платежи > Рекуррентные СБП-подписки [Получить подписку](https://docs.platega.io/%D0%BF%D0%BE%D0%BB%D1%83%D1%87%D0%B8%D1%82%D1%8C-%D0%BF%D0%BE%D0%B4%D0%BF%D0%B8%D1%81%D0%BA%D1%83-40029717e0.md): Возвращает подписку по указанному ID
- Платежи > Рекуррентные СБП-подписки [Список подписок](https://docs.platega.io/%D1%81%D0%BF%D0%B8%D1%81%D0%BE%D0%BA-%D0%BF%D0%BE%D0%B4%D0%BF%D0%B8%D1%81%D0%BE%D0%BA-40029720e0.md): Получить список ваших подписок
- Платежи > Рекуррентные СБП-подписки [Отменить подписку](https://docs.platega.io/%D0%BE%D1%82%D0%BC%D0%B5%D0%BD%D0%B8%D1%82%D1%8C-%D0%BF%D0%BE%D0%B4%D0%BF%D0%B8%D1%81%D0%BA%D1%83-40029730e0.md): Отмена останавливает будущие списания. Ручка идемпотентна. Плательщик также может отменить подписку сам — по ссылке из email, которое мы отправляем после каждого списания; вы узнаете об этом из SUBSCRIPTION_CANCELLED.
- Платежи > Рекуррентные СБП-подписки [Callback по списанию](https://docs.platega.io/callback-%D0%BF%D0%BE-%D1%81%D0%BF%D0%B8%D1%81%D0%B0%D0%BD%D0%B8%D1%8E-40029713e0.md): Приходит на каждое списание — успешное и неуспешное. Отличается от callback'а обычного платежа только двумя дополнительными полями: SubscriptionId и NextChargeAt.
- Платежи > Рекуррентные СБП-подписки [Callback по статусу подписки](https://docs.platega.io/callback-%D0%BF%D0%BE-%D1%81%D1%82%D0%B0%D1%82%D1%83%D1%81%D1%83-%D0%BF%D0%BE%D0%B4%D0%BF%D0%B8%D1%81%D0%BA%D0%B8-40030962e0.md): Приходит при смене статуса. В нём Id = SubscriptionId (ID подписки, не транзакции).
- Платежи [Создание платежной ссылки без заданного метода](https://docs.platega.io/%D1%81%D0%BE%D0%B7%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5-%D0%BF%D0%BB%D0%B0%D1%82%D0%B5%D0%B6%D0%BD%D0%BE%D0%B9-%D1%81%D1%81%D1%8B%D0%BB%D0%BA%D0%B8-%D0%B1%D0%B5%D0%B7-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE-%D0%BC%D0%B5%D1%82%D0%BE%D0%B4%D0%B0-33845703e0.md): Создает транзакцию и возвращает данные для оплаты. **ID транзакции генерируется системой автоматически — не передавайте поле `id` в запросе.**
- Платежи [Создание платежной ссылки с заданным методом](https://docs.platega.io/%D1%81%D0%BE%D0%B7%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5-%D0%BF%D0%BB%D0%B0%D1%82%D0%B5%D0%B6%D0%BD%D0%BE%D0%B9-%D1%81%D1%81%D1%8B%D0%BB%D0%BA%D0%B8-%D1%81-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D0%BC-%D0%BC%D0%B5%D1%82%D0%BE%D0%B4%D0%BE%D0%BC-29203843e0.md): Создает транзакцию и возвращает данные для оплаты. **ID транзакции генерируется системой автоматически — не передавайте поле `id` в запросе.**
- Платежи [Получение QR-кода для H2H-транзакции](https://docs.platega.io/%D0%BF%D0%BE%D0%BB%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D0%B5-qr-%D0%BA%D0%BE%D0%B4%D0%B0-%D0%B4%D0%BB%D1%8F-h2h-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B8-34794775e0.md): Возвращает QR-код или платежную ссылку
- Платежи [Проверка статуса оплаты платежа](https://docs.platega.io/%D0%BF%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%BA%D0%B0-%D1%81%D1%82%D0%B0%D1%82%D1%83%D1%81%D0%B0-%D0%BE%D0%BF%D0%BB%D0%B0%D1%82%D1%8B-%D0%BF%D0%BB%D0%B0%D1%82%D0%B5%D0%B6%D0%B0-29203844e0.md): Возвращает статус и детали транзакции.
- Платежи [Выгрузка транзакций в CSV](https://docs.platega.io/%D0%B2%D1%8B%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B9-%D0%B2-csv-37963792e0.md): Возвращает ссылку на CSV-файл с транзакциями по заданным фильтрам.

- Платежи [Выгрузка транзакций в Json](https://docs.platega.io/%D0%B2%D1%8B%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B9-%D0%B2-json-37991987e0.md): Возвращает ссылку на Json-файл с транзакциями по заданным фильтрам.

- Возвраты [Проверка возможности отмены транзакции](https://docs.platega.io/%D0%BF%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%BA%D0%B0-%D0%B2%D0%BE%D0%B7%D0%BC%D0%BE%D0%B6%D0%BD%D0%BE%D1%81%D1%82%D0%B8-%D0%BE%D1%82%D0%BC%D0%B5%D0%BD%D1%8B-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B8-38219023e0.md): Возвращает информацию о том, доступна ли отмена для указанной транзакции, и какая сумма в USDT будет списана с баланса при её проведении.
- Возвраты [Отмена транзакции](https://docs.platega.io/%D0%BE%D1%82%D0%BC%D0%B5%D0%BD%D0%B0-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B8-38225949e0.md): Инициирует отмену транзакции и возврат средств плательщику. Перед вызовом рекомендуется проверить возможность отмены через cancel-supported.
- Выводы [Получение сохранённых карт](https://docs.platega.io/%D0%BF%D0%BE%D0%BB%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D0%B5-%D1%81%D0%BE%D1%85%D1%80%D0%B0%D0%BD%D1%91%D0%BD%D0%BD%D1%8B%D1%85-%D0%BA%D0%B0%D1%80%D1%82-39075563e0.md): По умолчанию возвращаются только активные карты. Передайте onlyActive=false чтобы получить также карты со статусом DISABLED и PENDING.
- [Callback об изменении статуса транзакции](https://docs.platega.io/callback-%D0%BE%D0%B1-%D0%B8%D0%B7%D0%BC%D0%B5%D0%BD%D0%B5%D0%BD%D0%B8%D0%B8-%D1%81%D1%82%D0%B0%D1%82%D1%83%D1%81%D0%B0-%D1%82%D1%80%D0%B0%D0%BD%D0%B7%D0%B0%D0%BA%D1%86%D0%B8%D0%B8-29209725e0.md): Ваш endpoint для приема callback. Укажите URL в ЛК (Настройки → Callback URLs). Поставщик отправляет заголовки `X-MerchantId` и `X-Secret` и JSON-тело. При успешной оплате — статус **CONFIRMED**, при неуспешной — **CANCELED**, при возврате денежных средств — **CHARGEBACKED**. В случае отсутствия успешного ответа в течение 60 секунд запрос отменяется, затем выполняется до 3 повторных попыток с интервалом 5 минут.
